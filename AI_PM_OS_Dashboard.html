    🧠 RCGCF 公式详解

    字母含义示例
    Role角色"你是一位高级AI产品经理"
    Context背景"我们正在开发一个智能客服系统"
    Goal目标"请编写一份完整的PRD文档"
    Constraint约束"输出不超过2000字，包含用户故事"
    Format格式"使用Markdown格式，分章节呈现"
    RCGCF = Role + Context + Goal + Constraint + Format

    🏅 分级练习体系

    入门级 基础 Prompt 编写

    掌握 RCGCF 结构，能写出清晰的单轮 Prompt

    进阶级 多轮对话设计

    设计复杂的对话流，处理边界情况

    高级 System Prompt 架构

    设计完整的 System Prompt，包含 Few-shot、CoT 等

    ❌ vs ✅ Prompt 对比案例

    ❌ 差 Prompt：
    帮我写个产品需求文档
    问题：无角色、无背景、无目标、无约束、无格式 → AI 只能猜测
    ✅ 好 Prompt：
    [Role] 你是一位有 8 年经验的 B2B SaaS 产品经理，擅长企业级产品设计。
    [Context] 我们公司要开发一个 AI 智能客服系统，服务于电商行业的中大型客户。
    [Goal] 请编写一份 PRD，覆盖核心功能需求和非功能性需求。
    [Constraint] 字数控制在 3000 字以内，必须包含用户故事格式。
    [Format] 使用 Markdown，分为：产品概述、用户画像、功能列表、用户故事、验收标准。

    🔗 RAG (检索增强生成)

    什么是 RAG？
    RAG = Retrieval-Augmented Generation，结合了检索和生成的 AI 架构。

    核心原理：
    1. 用户提问 → 2. 向量数据库检索相关文档 → 3. 将文档作为上下文 → 4. LLM 生成回答并引用来源

    典型应用：企业知识库问答、智能客服、法律咨询、医疗诊断辅助
    RAG 核心组件
    • Embedding Model: 将文本转换为向量 (OpenAI ada-002, Cohere)
    • Vector Database: 存储和检索向量 (Pinecone, Weaviate, Milvus)
    • Retriever: 相似度搜索算法 (BM25, Dense Retrieval)
    • Generator: LLM 生成最终回答 (GPT-4, Claude)
    作为 PM 的关键决策点
    • 选择哪个 Embedding 模型？（精度 vs 成本）
    • 向量数据库选型？（云服务 vs 自建）
    • 如何保证数据新鲜度？（增量更新策略）
    • 如何评估 RAG 质量？（RAGAS 框架）

    🤖 Agent (智能体)

    什么是 AI Agent？
    Agent = 具备自主决策和行动能力的 AI 系统，不只是"回答问题"，而是"完成任务"。

    核心特征：
    1. 感知 - 接收和理解环境信息
    2. 推理 - 规划任务步骤
    3. 行动 - 调用工具/API 执行
    4. 反思 - 根据结果调整策略
    主流 Agent 框架
    • LangChain / LangGraph - 最流行的 Agent 开发框架
    • CrewAI - 多 Agent 协作框架
    • AutoGPT - 自主 Agent 先驱
    • OpenAI Assistants API - 官方 Agent 方案
    • n8n - 低代码工作流 Agent
    Agent 设计模式
    • ReAct - Reasoning + Acting 循环
    • Plan-and-Execute - 先规划后执行
    • Multi-Agent - 多个专业 Agent 协作
    • Tool Use - 动态选择和调用工具

    🔌 MCP (Model Context Protocol)

    什么是 MCP？
    MCP = Model Context Protocol，Anthropic 提出的开放标准，让 AI 模型能够安全地与外部数据源和工具交互。

    类比理解：MCP 对 AI 就像 USB 对电脑 - 统一的接口标准

    核心价值：一次集成，所有 MCP 兼容的工具都能用
    MCP 三层架构
    • MCP Client - AI 应用 (Claude Desktop, IDE 插件)
    • MCP Server - 数据源/工具 (GitHub, Slack, 数据库)
    • MCP Host - 运行环境 (桌面应用, Web 应用)
    为什么 PM 要关注 MCP？
    • 降低 AI 集成成本 - 不再需要逐个对接 API
    • 生态效应 - 越多的 MCP Server，产品越强大
    • 标准化竞争 - 类似 App Store 的平台机会
    • 未来趋势 - 可能成为 AI 时代的"API 标准"
